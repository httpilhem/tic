SYSTEME DE GESTION DE VERSION


    Git:

est un system de gestion de version les plus populaires qui permet aux utilisateurs de travailler sur une même version de code source.

    Modèles de logiciel de version :

Modèle centralisé :

leur principe est la centralisation de code source lie au projet

-Modèle distribue :

le code source de projet est toujours héberger sur un serveur distant et chaque utilisateur est invité télécharger et héberger l intégralité du code source.

GitHub:

est une plateforme de développement de logiciels base sur git.

Utiliser git :

nous allons utiliser la console et donc un langage en lignes de commande

-utiliser la ligne de commandes et le seul moyen d’accès a toutes les commandes git.

« Git-config --global user.name”salsabil ilhem” » et git config –global user.email « votre emali » a la suite pour enseigner un nom et une adresse email. Et pour assurer que les informations ont bien été enregistrées. On tape : “Git config user.name “ et “git config user.email “

Les états des fichiers :

  • Modifier (“modified”)
  • Indexer(“staged”)
  • Valider(“commited”)

    • Les zones du travail :

    Chaque projet Git composé de trois sections :

  • Working tree
  • staged area
  • repository
      • -“working tree” correspond d une extraction unique “chek out”
    -“staging area”correspond à un simple fichier -le répertoire git est ou il stocke les métadonnées (le cœur de Git).

    Créer un dépôt Git à partir d’un répertoire existant :

    Je vais créer un répertoire « projet Git » qui se trouve sur mon bureau et qui contient deux fichiers textes vides “fichier1.txt” et “READ ME.txt”.

      pour initialiser un dépôt git on utilise la commande “git init” on peut utiliser la commande “git satuts“ pour déterminer l’état des fichiers de notre répertoire.
      Pour indexer les fichiers on utilise la commande “git commit”.

    Cloner un dépôt Git :
